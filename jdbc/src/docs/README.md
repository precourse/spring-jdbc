# 미션

JDBC 라이브러리 구현하기

### 목표

- SQL 실행과 매핑을 위한 공통 템플릿을 만들어 DAO의 JDBC 중복 코드를 줄인다.

### step1

1. JdbcTemplate을 구현해 update, query, queryForObject를 제공한다.
2. 조회 매핑을 위해 RowMapper를 사용한다.
3. JDBC 자원 관리와 예외 처리 방식을 템플릿으로 일원화한다.
4. UserDao는 SQL 작성/파라미터 전달/매핑만 담당하게 구조를 변경한다.

### step2

1. 조회 동작의 일관성을 높인다(단건/없음/다건 케이스 처리).
2. 반환/예외 규격을 명확히 정리한다(Optional/단건 초과 예외 등).
3. 코드 구조를 정돈해 사용성을 개선한다.

### step3

1. 사용자 변경 시나리오에 트랜잭션 경계를 설정한다.
2. 트랜잭션 동기화를 통해 같은 단위에서 동일 커넥션이 사용되게 한다.
3. 트랜잭션 처리와 비즈니스 로직을 분리한다.
4. 실패 시 롤백이 동작해야 한다.
